## Context

当前背景视觉系统由三层组成：Canvas 粒子网络（BlueprintGrid）、CSS 极光光斑（AuroraGlow）、CSS 纹理叠加（film grain + scanlines）。背景色为 `midnight #0b0d17`。问题在于所有动态元素的 opacity 偏低，暗色背景吸收了大部分视觉信息，导致"科技感"在实际观感中打折扣。

## Goals / Non-Goals

**Goals:**
- 背景色微微提亮，拉开层次空间
- 粒子网络连线和点 opacity 提升 2-3 倍
- 网格线更明显、更密集
- 浮游几何体更可见
- 新增数据脉冲光点沿连线穿梭
- 保持暗色调氛围和整体美学一致

**Non-Goals:**
- 不改变页面布局或内容卡片样式
- 不引入 Three.js 或 WebGL 等新依赖
- 不修改 AuroraGlow 的动画逻辑（仅调参）

## Decisions

### 1. 背景色改为 `#0d1117`
`#0b0d17` → `#0d1117`，仅提亮 2 个亮度点。保持暗色调但让极暗区域（近乎纯黑）有更多层次感。所有其他颜色不变。

### 2. 粒子网络参数调整（全部在 BlueprintGrid.astro 内）
| 参数 | 旧值 | 新值 |
|------|------|------|
| GRID_SPACING | 80px | 60px |
| 网格线 strokeStyle | rgba(255,255,255,0.025) | rgba(255,255,255,0.05) |
| 普通点 alpha 范围 | 0.05-0.15 | 0.12-0.30 |
| 节点 alpha | 0.5 | 0.7 |
| 连线 baseAlpha | 0.12 | 0.25 |
| 连线距离 | 150px | 180px |
| 几何体 alpha | 0.02-0.03 | 0.06-0.10 |
| GEO_COUNT | 5 | 5 (不变) |

### 3. 数据脉冲实现
在 draw() 循环中维护 3-5 个 pulse 对象，每个绑定一条随机连线（两个粒子索引 + 进度 t 0→1）。每帧 t += speed，到达 1 时重新选择一条连线。绘制时沿连线的两个端点做线性插值位置，以径向渐变 + 外层光晕渲染光点，alpha 0.5-0.8。

### 4. Aurora 微调
四个光斑的 opacity 从 0.08-0.12 → 0.10-0.16，配合提亮后的背景保持相同的视觉对比度。

## Risks / Trade-offs

- [粒子连线 O(n²) 检查]：100 个点两两距离检查 = ~5000 次/帧。当前帧率稳定，无需改为空间哈希。若未来增加粒子数，优先改为网格分桶。
- [移动端降级]：保持现有 isMobile 分支——粒子数 40、连线距离 120px、脉冲数 1-2、无 cursor glow，确保移动端性能。
